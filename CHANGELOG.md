### 计划新增
* 接入DeepSeek API或OpenAI API
* 新增大模型配置项`LLM_CONFIG`
* 新增API调用能力
* 新增大模型识别用户自然语言意图
* 使用大模型提取结构化差旅信息
* 控制大模型返回JSON格式结果
* 解析大模型返回的结构化数据
* 使用大模型生成更自然的用户回复
* 增加大模型调用异常处理逻辑
* 在大模型调用失败时，支持返回兜底回复或退出规则识别流程
* 为后续`OpenAIChatModel`，`IntentionAgent`和多Agent调度能力打基础

### notes

本阶段的目标是让系统从规则助手升级为具备基础自然语言理解能力的智能差旅助手。

在前几个版本中，系统主要依靠关键词规则完成意图识别、结构化信息提取、任务调度、偏好记忆和费用记录。虽然系统已经可以跑通基础流程，但规则方式存在明显局限：用户表达稍微变化，系统就容易识别错误。

本阶段开始接入大模型能力，让大模型负责理解用户自然语言输入，并将用户需求转换为程序可以处理的结构化JSON数据

当前计划中的系统主流程为：

用户输入

⬇

识别用户同意图

⬇

提取结构化信息

⬇

返回JSON格式结果

⬇

系统解析JSON

⬇

读取用户偏好/费用记录/行程信息

⬇

任务调度

⬇

行程规划/偏好记忆/费用记录/报销清单/普通聊天

⬇

返回给用户

## [0.7.1] - 2026-07-01

### added
* 将意图识别逻辑从`main.py`拆分到独立模块
* 将结构化信息提取逻辑从`main.py`拆分到独立模块
* 将行程规划逻辑从`main.py`拆分到独立模块
* 将偏好记忆逻辑从`main.py`拆分到独立模块
* 将任务调度逻辑从`main.py`拆分到独立模块
* 保留`main.py`作为程序入口
* 降低单文件复杂度，提高代码可维护性

### notes

本阶段目标不是新增业务功能，而是整理代码结构

随着系统已经完成输入输出、意图识别、结构化信息提取、任务调度、行程规划和偏好记忆，main.py中承担的职责越来越多。为了让项目继续扩展，需要进行一次模块化重构。

完成后，项目结构将从单文件模式升级为多模块结构：

main.py

intent.py

extractor.py

planner.py

preference.py

dispatcher.py

本阶段对应真实项目中从“功能堆叠”走向“模块化架构”的过程

## [0.6.0] - 2026-06-30

### Added

- 优化 `plan_trip()` 行程规划函数
- 新增交通建议生成函数 `generate_transport_advice()`
- 新增住宿建议生成函数 `generate_hotel_advice()`
- 新增日程建议生成函数 `generate_schedule_advice()`
- 新增预算提醒生成函数 `generate_budget_advice()`
- 新增约束提醒生成函数 `generate_constraint_advice()`
- 支持根据结构化差旅信息生成基础行程方案
- 支持输出交通、住宿、日程、预算和约束提醒

### notes

本阶段目标是让系统具备真正的基础行程规划能力。

在mvp3中，系统已经可以根据用户意图调用`plan_trip()`，但是此时的行程规划仍然比较基础。mvp4要进一步增强`plan_trip()`，让他能够根据结构化差旅信息生成更完整的差旅安排。

系统已经可以从用户自然语言输入中提取结构化差旅信息，并根据出发地、目的地、出行时间、预算、偏好和约束条件生成一个基础可用的差旅行程方案。

完成后，系统主流程将升级为：

用户输入

⬇

意图识别

⬇

结构化信息提取

⬇

任务调度

⬇

生成基础差旅方案

⬇

返回给用户
